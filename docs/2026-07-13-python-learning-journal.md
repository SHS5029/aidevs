# 2026-07-13 Python 학습 일지

## 오늘의 학습 목표

오늘은 Python의 주요 자료구조를 직접 조작하고, 반복되는 코드를 함수로 분리하는 연습을 했다. 단순히 문법을 따라 쓰는 데서 끝내지 않고 학생 점수와 중첩 API 응답처럼 실제 데이터와 비슷한 구조에 적용하는 것이 목표였다.

## 1. 자료구조 다루기

`list`에서는 `append()`, `insert()`, 인덱싱, 값 수정, `pop()`, `len()`을 실습했다. `for`로 값을 하나씩 꺼내고 `sum()`과 `len()`으로 합계와 평균을 계산했으며, 짝수만 골라 합계와 평균을 구할 때는 조건을 만족한 항목의 개수를 별도로 세어야 한다는 점을 배웠다.

`tuple`은 값을 변경할 수 없는 순서형 자료구조이고, `name, score = ("Jean", 95)`처럼 언패킹할 수 있다는 것을 확인했다. 함수에서 여러 값을 반환하면 실제로는 튜플로 묶이며, 반환값을 각각의 변수에 나누어 담거나 반복문으로 처리할 수 있었다.

`dict`에서는 이름과 과목별 점수를 key-value 형태로 저장하고, 점수의 합계와 평균을 계산해 `sum`, `avg` 키로 다시 추가했다. `keys()`와 `values()`의 결과를 list나 tuple로 변환하는 방법도 실습했다. 여러 학생은 `list[dict]` 구조로 표현하여 반복문 하나로 이름과 점수를 출력하고 전체 평균을 계산했다.

`set`에서는 중복 태그 제거, 값 추가·삭제, `in`을 이용한 포함 여부 확인을 연습했다. 학생마다 가진 태그를 하나의 set에 `update()`하면 중복 없는 전체 태그 목록을 만들 수 있었다.

## 2. 중첩 데이터 탐색

API 응답 형태의 dict 안에서 `status`, `count`, `data`를 꺼내고, `data` 안의 각 메시지와 `tags`에 접근했다. 중첩 구조는 바깥 key부터 한 단계씩 변수에 담으면 이해하기 쉬웠다. 각 메시지의 태그 개수는 `len()`으로, 전체 태그 개수는 `sum(len(message["tags"]) for message in data_list)`와 같은 표현으로 계산했다.

## 3. 함수의 역할과 재사용

`def`로 함수를 정의하고 docstring, 매개변수, 반환값, 타입 힌트를 작성했다. 출력만 하는 함수와 값을 `return`하는 함수의 차이도 확인했다. 반환값이 있으면 이후 계산이나 다른 함수에서 다시 사용할 수 있어 코드의 재사용성이 높아졌다.

기본값 매개변수와 키워드 인자를 사용해 호출 방법을 바꾸어 보았고, 사용자 정보를 dict로 반환하는 함수도 작성했다. 합계와 평균, 최솟값과 최댓값을 함수로 묶고 여러 데이터에 반복 적용하면서 같은 계산 코드를 복사하지 않아도 된다는 장점을 체감했다.

계산기 실습에서는 연산자에 따라 계산을 분기하고, 0으로 나누는 경우와 잘못된 연산자를 처리했다. 숫자 목록의 평균보다 큰 값만 반환하는 함수에서는 먼저 평균 계산 함수를 호출한 뒤 결과를 필터링했다. 작은 함수를 조합하면 복잡한 작업도 단계별로 표현할 수 있었다.

## 4. 학생 데이터 과제로 종합하기

학생 목록을 대상으로 등급 판정, 통과 여부 확인, 평균 계산, 통과 학생 필터링, 학생 정보 출력을 각각 함수로 분리했다. `is_passed()`를 필터 함수에서 다시 사용하고, 점수 목록을 만들어 `calculate_average()`에 전달했다. 평균 미만 학생을 찾는 추가 기능까지 구현하면서 자료구조, 반복문, 조건문, 함수가 하나의 흐름으로 연결되는 것을 확인했다.

같은 구조를 다른 문제에도 적용할 수 있도록 상품 재고 관리 연습 문제도 준비했다. 학생의 점수와 통과 여부를 상품의 재고 수량과 구매 가능 여부로 바꾸면 기존 함수 설계 방식을 새로운 데이터에 재사용할 수 있다.

## 시행착오와 보완할 점

- `enumerate()`에는 `start`만 지정할 수 있고 `step` 인자는 없으므로, 일정한 간격이 필요하면 별도의 계산이 필요하다.
- 0으로 나눌 때 `None`을 반환하는 함수는 반환 타입도 `float | None`으로 표시해야 실제 동작과 타입 힌트가 일치한다.
- 빈 list의 평균을 계산하면 0으로 나누는 문제가 생길 수 있으므로, 입력 조건을 정하거나 빈 목록 처리 방법을 결정해야 한다.
- `dict[str, object]`는 여러 자료형을 담기 편하지만 특정 값이 숫자라는 정보가 약하다. 데이터 구조가 복잡해질수록 더 구체적인 타입 표현을 공부할 필요가 있다.
- 함수 이름은 `add_numbers`, `calculate_sum_and_average`처럼 역할이 바로 드러나도록 짓고 오타를 줄여야 한다.
